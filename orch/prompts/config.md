# 설정 관리 프롬프트

## 입력

```
설정 요청: {{config_request}}
현재 설정: {{current_settings}}
```

## 지시사항

시스템 프롬프트에 정의된 역할과 규칙을 따라 `{{config_request}}`를 처리하세요.

### Step 1: 요청 유형 판별

`{{config_request}}`에서 다음 중 하나를 판별하세요:

- **조회(get)**: 현재 설정 확인 요청
- **변경(set)**: 설정 값 변경 요청
- **검증(validate)**: 설정 유효성 확인 요청

### Step 2: 설정 대상 식별

시스템 프롬프트 **"핵심 역량"** (스킬 관리, 산출물 출력 경로, 파이프라인 템플릿, 에이전트 설정, 프로필) 및 **"프로필 기반 설정"** 표에 정의된 항목을 기준으로 대상을 식별합니다.

### Step 3: 변경 검증

시스템 프롬프트 **"설정 검증"** 기준에 따라 다음을 확인합니다:

1. 경로 유효성 (존재/쓰기 권한)
2. 스킬 의존성 충돌
3. 파이프라인 호환성
4. 값 범위

### Step 4: 결과 출력

시스템 프롬프트 **"출력"** 스키마(`config_result`)에 따라 결과를 출력합니다. 저장 위치는 시스템 프롬프트 **"설정 저장 위치"** 참조.

시스템 프롬프트 **"출력 프로토콜"**에 따라 `meta.json`/`body.md`에 기록합니다.
