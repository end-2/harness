# 디스패치 프롬프트

## 입력

```
사용자 요청: {{user_request}}
현재 실행 상태: {{current_run_state}}
출력 루트: {{output_root}}
```

## 지시사항

시스템 프롬프트에 정의된 역할과 규칙을 따라 `{{user_request}}`를 분석하여 적절한 파이프라인 또는 스킬로 라우팅하세요.

### Step 1: 현재 상태 확인

`{{current_run_state}}`를 확인하여 상태를 파악합니다:

- `status: idle` → 새로운 파이프라인 시작 가능
- `status: running` → 활성 실행 존재 — 재개/새 실행 판단 필요
- 상태 파일 없음 → 첫 실행으로 간주

### Step 2: 의도 분석

시스템 프롬프트 **"의도 분석 및 라우팅"** 및 **"기존 프로젝트 감지"**, **"재개 요청 인식"** 기준에 따라 작업 유형, 기존 프로젝트 여부, 프로젝트 경로, 재개 여부를 판별합니다.

### Step 3: 파이프라인 선택

시스템 프롬프트 **"파이프라인 선택 매트릭스"** 표에 따라 파이프라인을 선택합니다.

### Step 4: 선행 조건 검증

시스템 프롬프트 **"선행 조건 검증"** 기준으로 경로/run/스킬 활성화를 확인합니다. 의도가 불명확하면 짧은 확인 질문을 합니다.

### Step 5: 결과 출력

시스템 프롬프트 **"출력"** 스키마(`dispatch_result` — new_pipeline / resume / single_skill)에 따라 결과를 출력합니다. `parameters.output_root`에는 `{{output_root}}`를 사용합니다.

시스템 프롬프트 **"출력 프로토콜"**에 따라 `meta.json`/`body.md`에 기록합니다.
