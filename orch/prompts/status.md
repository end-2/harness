# 현황 조회 프롬프트

## 입력

```
조회 요청: {{status_request}}
출력 루트: {{output_root}}
```

## 지시사항

시스템 프롬프트에 정의된 역할과 규칙을 따라 `{{status_request}}`를 처리하세요.

### Step 1: 조회 유형 판별

시스템 프롬프트 **"핵심 역량"**에 정의된 유형 중 하나로 판별합니다: `skills` / `runs` / `run_detail` / `search` / `stats`.

### Step 2: 데이터 수집

시스템 프롬프트 **"데이터 소스"** 절에 명시된 경로에서 데이터를 수집합니다:

- 스킬 목록: 각 스킬의 `skills.yaml`
- 실행 이력/상세: `{{output_root}}/runs/*/run.meta.md`
- 산출물 검색: `{{output_root}}/runs/*/<skill>/*.md`
- 현재 상태: `{{output_root}}/current-run.md`

각 조회 유형의 상세 포맷은 시스템 프롬프트 **"스킬 목록 조회"**, **"실행 이력 조회"**, **"특정 Run 상세 조회"**, **"산출물 검색"**, **"사용 통계"**, **"스킬 의존성 시각화"** 절 참조.

### Step 3: 결과 포맷팅

시스템 프롬프트의 각 조회 유형 예시에 맞춰 마크다운 테이블/목록으로 포맷팅합니다. 대량 결과는 최근 항목 위주로 요약합니다.

### Step 4: 결과 출력

시스템 프롬프트 **"출력"** 스키마(`status_result`)에 따라 결과를 출력합니다. 존재하지 않는 run-id/스킬 요청은 해당 항목 없음을 안내합니다.

시스템 프롬프트 **"출력 프로토콜"**에 따라 `meta.json`/`body.md`에 기록합니다.
