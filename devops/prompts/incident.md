# 인시던트 대응 프롬프트

## 입력

```
Strategy 산출물: {{strategy_output}}
Monitor 산출물: {{monitor_output}}
Arch 산출물: {{arch_output}}
IaC 산출물: {{iac_output}}
```

## 지시사항

당신은 인시던트 대응 전문가입니다. 배포 전략의 롤백 절차와 모니터링 알림 규칙을 분석하여 인시던트 대응 런북을 자동 생성하세요.

### Step 1: 알림 규칙 분석

Monitor 산출물의 `monitoring_rules`에서:

- 모든 알림 규칙의 `id`, `condition`, `severity`를 추출
- 각 알림이 어떤 SLO와 연결되어 있는지 `slo_refs`를 확인
- 심각도별로 분류

### Step 2: 롤백 절차 분석

Strategy 산출물에서:

- `rollback_trigger`: 자동 롤백이 발동하는 조건
- `rollback_procedure`: 롤백 절차의 각 단계
- `health_checks`: 헬스 체크 경로와 조건

### Step 3: 장애 시나리오 도출

Arch `component_structure`의 컴포넌트 유형별 주요 장애 시나리오를 도출하세요:

- `service`: 높은 레이턴시, 5xx 급증, OOM, 의존성 서비스 장애
- `store` (SQL): 연결 풀 소진, 슬로우 쿼리, 레플리케이션 지연, 디스크 부족
- `store` (cache): 캐시 미스 급증, 메모리 부족, 클러스터 분리
- `queue`: 메시지 적체, 소비자 지연, 데드 레터 큐 증가
- `gateway`: 인증 실패 급증, 레이트 리밋, 라우팅 오류, SSL 인증서 만료

### Step 4: 알림별 런북 생성

각 알림 규칙에 대응하는 런북을 생성하세요. 각 런북은:

1. **트리거 조건**: 어떤 알림이 발생했을 때 이 런북을 실행하는가
2. **증상**: 관찰 가능한 증상 체크리스트
3. **진단 절차**: 구체적 명령어와 쿼리 포함
4. **조치 절차**: 자동 조치(이미 설정된 것)와 수동 조치 구분
5. **에스컬레이션**: 심각도별 담당자, 채널, 시간 기준

### Step 5: 진단 명령어 생성

IaC 산출물에서 인프라 접근 방법을 추출하여 구체적 명령어를 포함하세요:

```bash
# 서비스 로그 확인
kubectl logs -n [namespace] -l app=[service] --since=5m

# 리소스 상태 확인
kubectl top pods -n [namespace] -l app=[service]

# 데이터베이스 연결 상태
psql -h [host] -U [user] -c "SELECT * FROM pg_stat_activity;"

# 메시지 큐 상태
aws sqs get-queue-attributes --queue-url [url] --attribute-names All
```

### Step 6: 에스컬레이션 경로 설정

심각도별 에스컬레이션 매트릭스를 정의하세요:

| 심각도 | 초기 대응 | 15분 미해결 | 30분 미해결 | 1시간 미해결 |
|--------|----------|-----------|-----------|------------|
| `critical` | 온콜 + 팀 리드 | 엔지니어링 매니저 | VP | CTO |
| `high` | 온콜 | 팀 리드 | 매니저 | - |
| `medium` | 담당 개발자 | 팀 리드 | - | - |
| `low` | 백로그 등록 | - | - | - |

### Step 7: 커뮤니케이션 템플릿

내부 공지, 외부 상태 페이지, 사후 분석 템플릿을 생성하세요.

### Step 8: 산출물 정리

다음 형식으로 산출물을 정리하세요:

**런북 목록**: ID, 제목, 트리거, 심각도, 관련 알림, 관련 SLO
**에스컬레이션 매트릭스**: 심각도, 초기, 15분, 30분, 1시간
**커뮤니케이션 템플릿**: 내부 공지, 외부 상태, 사후 분석

## 주의사항

- 모든 알림 규칙에 대응하는 런북이 있어야 합니다 (1:1 매핑)
- 진단/조치 명령어는 실제 실행 가능한 수준이어야 합니다
- 자동 롤백이 설정된 경우에도 수동 롤백 절차를 포함하세요 (자동 실패 대비)
- 에스컬레이션 경로는 명확한 시간 기준과 담당자를 포함해야 합니다
