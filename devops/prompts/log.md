# 로깅 프롬프트

## 입력

```
Arch 산출물: {{arch_output}}
IaC 산출물: {{iac_output}}
```

## 지시사항

시스템 프롬프트에 정의된 역할과 규칙을 따라 Arch 컴포넌트 구조와 보안 제약을 분석하여 구조화된 로깅 표준과 설정을 자동 생성하세요.

### Step 1: 컴포넌트 분석

Arch `component_structure`에서 컴포넌트 이름·유형·인터페이스와 `dependencies`(서비스 간 호출 경로)를 추출합니다.

### Step 2: 구조화 로깅 표준 정의

시스템 프롬프트 **"구조화 로깅 표준"**의 JSON 포맷과 필수 필드를 그대로 적용합니다.

### Step 3: 컴포넌트 유형별 로깅 전략

시스템 프롬프트 **"컴포넌트 유형별 로깅 전략"** 표에 따라 각 컴포넌트의 추가 로그 항목을 정의합니다.

### Step 4: 로그 레벨 가이드라인

시스템 프롬프트 **"로그 레벨 가이드라인"** 표를 참조하여 환경별 로그 레벨 정책을 설정합니다.

### Step 5: 상관 ID 전파 설계

Arch `component_structure.dependencies`를 따라 시스템 프롬프트 **"상관 ID 전파"** 절의 프로토콜별 전파 방식(HTTP/큐/gRPC)을 적용하고, 진입점(gateway)에서 생성하여 이후 전파합니다.

### Step 6: 민감 정보 마스킹

RE `constraints`의 regulatory 제약을 확인하고, 시스템 프롬프트 **"민감 정보 마스킹"** 표의 필드 유형별 마스킹 규칙을 적용합니다.

### Step 7: 보존 및 로테이션 정책

시스템 프롬프트 **"보존 및 로테이션 정책"** 표에 따라 환경별 보존 기간과 아카이브 정책을 설정합니다. 컴플라이언스 요구가 있으면 우선 적용합니다.

### Step 8: 로그 기반 메트릭

시스템 프롬프트 **"Monitor 연동: 로그 기반 메트릭"** 표에 따라 monitor 에이전트에 제공할 메트릭을 정의합니다.

### Step 9: 산출물 정리

시스템 프롬프트 **"출력 형식"** 표(로깅 설정 요약 / 컴포넌트별 로깅 전략 / 민감 정보 마스킹 규칙 / 로그 기반 메트릭)에 맞춰 정리하고, **"출력 프로토콜"**에 따라 `meta.json`/`body.md`에 기록합니다. 컴플라이언스-성능 충돌 시 시스템 프롬프트 **"에스컬레이션 조건"**을 따릅니다.
