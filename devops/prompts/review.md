# DevOps 리뷰 프롬프트

## 입력

```
전체 DevOps 산출물: {{all_devops_outputs}}
Arch 산출물: {{arch_output}}
Impl 산출물: {{impl_output}}
```

## 지시사항

시스템 프롬프트에 정의된 역할과 규칙을 따라 전체 DevOps 산출물의 정합성과 완전성을 검증하고, 자동 수정 가능한 이슈는 해당 에이전트에 피드백하세요.

### Step 1: 배포-관찰 피드백 루프 검증

시스템 프롬프트 **"배포-관찰 피드백 루프 검증"** 표의 6개 연결(Strategy→Monitor, Monitor→Incident, SLO→Strategy 연결/정책, Log→Monitor, Strategy→Incident)을 모두 검증합니다. 누락은 표의 "실패 시" 열에 따라 자동 수정합니다.

### Step 2: 선행 스킬 추적성 검증

시스템 프롬프트 **"선행 스킬 추적성 검증"** 표의 5개 항목을 검증합니다. Arch→IaC, RE→SLO 위반은 에스컬레이션, 나머지(Impl→Pipeline, Arch refs, Impl refs)는 자동 수정을 시도합니다.

### Step 3: 보안 베스트 프랙티스 검증

시스템 프롬프트 **"보안 베스트 프랙티스 검증"** 표(시크릿 노출 / 권한 최소화 / 이미지 보안 / 네트워크 보안 / 로그 마스킹)의 항목을 검증합니다.

### Step 4: 비용 최적화 리뷰

시스템 프롬프트 **"비용 최적화 리뷰"** 표(리소스 사이징 / 환경별 차등 / 예약 인스턴스 / 스팟 인스턴스 / 스토리지 계층)의 항목을 검토합니다.

### Step 5: 환경 일관성 검증

시스템 프롬프트 **"환경 일관성 검증"** 표(IaC 구조 동일성 / 설정 드리프트 / 시크릿 관리)의 항목을 검증합니다.

### Step 6: 컨테이너 이미지 최적화 (해당 시)

시스템 프롬프트 **"컨테이너 이미지 최적화"** 표를 적용합니다.

### Step 7: 자동 수정 처리

자동 수정 가능한 이슈는 이슈 식별 → 수정 방안 결정 → 해당 에이전트에 수정 요청 → 결과 확인 → 내역 기록의 순서로 처리합니다.

### Step 8: 산출물 정리

시스템 프롬프트 **"출력 형식"** 표(리뷰 요약 / 자동 수정 내역 / 에스컬레이션 항목 / 최적화 제안)에 맞춰 정리하고, **"출력 프로토콜"**의 승인 판정 단계(`--verdict APPROVED|CONDITIONAL|REJECTED`)까지 수행하여 `meta.json`/`body.md`에 기록합니다. 계약 위반 이슈는 시스템 프롬프트 **"에스컬레이션 조건"**에 따라 사용자에게 보고합니다.
