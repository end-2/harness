# 보안 코드 리뷰 프롬프트

## 입력

```
구현 맵: {{implementation_map}}
코드 구조: {{code_structure}}
컴포넌트 구조: {{component_structure}}
위협 모델: {{threat_model}}
취약점 보고서: {{vulnerability_report}}
```

## 지시사항

시스템 프롬프트에 정의된 역할과 규칙을 따라 threat-model의 위협과 audit의 취약점을 기반으로, 보안 로직의 정확성을 심층 리뷰하세요. audit이 패턴 매칭으로 탐지할 수 없는 **로직 레벨 보안 이슈**에 집중합니다. 사용자에게 질문하지 말고 자동으로 실행합니다.

### Step 1: 대응 전략 구현 검증

`threat_model`의 각 위협(`TM-xxx`)에 대해 `mitigation`이 코드에서 어떻게 구현되었는지 검증하세요:

1. `threat_model` 순회 → 각 `mitigation` 추출
2. `implementation_map`에서 관련 코드 위치 식별
3. 구현 상태 판정 (`implemented | partial | missing`)

시스템 프롬프트 **"리뷰 영역 → 1. 위협 대응 전략 구현 검증"** 표의 대응 전략별 검증 포인트를 적용합니다. 결과는 시스템 프롬프트 **"산출물 구조 → 2. 대응 전략 구현 매트릭스"** 스키마에 기록합니다.

**에스컬레이션 판단**: `risk_level: critical` 위협의 `implementation_status`가 `partial` 또는 `missing`이면 즉시 에스컬레이션합니다.

### Step 2: 인증 로직 심층 리뷰

`component_structure`에서 `type: gateway` 또는 인증 관련 인터페이스를 식별하고, 시스템 프롬프트 **"리뷰 영역 → 2. 인증/인가 로직 검증 → 인증(Authentication)"** 체크리스트를 적용합니다.

### Step 3: 인가 로직 심층 리뷰

시스템 프롬프트 **"리뷰 영역 → 2. 인증/인가 로직 검증 → 인가(Authorization), 세션 관리"** 체크리스트에 따라 수직/수평 권한 상승 방어와 세션 관리를 검증합니다.

### Step 4: 입력 검증 심층 리뷰

`component_structure.interfaces`에서 모든 `inbound` 인터페이스를 식별하고, 시스템 프롬프트 **"리뷰 영역 → 3. 입력 검증 및 새니타이징 검증"** 기준을 적용합니다.

### Step 5: 에러 핸들링 및 정보 노출 리뷰

시스템 프롬프트 **"리뷰 영역 → 4. 에러 핸들링의 정보 노출 방지"** 체크리스트를 적용합니다.

### Step 6: 보안 헤더 및 CORS 리뷰

시스템 프롬프트 **"리뷰 영역 → 5. 보안 헤더 및 CORS 설정"** 기준을 적용합니다.

### Step 7: 암호화 사용 적절성 리뷰

시스템 프롬프트 **"리뷰 영역 → 6. 암호화 사용 적절성"** 기준을 적용합니다.

### Step 8: 결과 구성

시스템 프롬프트 **"산출물 구조 → 1. 보안 리뷰 리포트"** 스키마에 따라 `security_review`를 구성하세요. audit이 이미 보고한 항목은 `vuln_refs`로 참조만 하고 중복 보고하지 않습니다.

**에스컬레이션 판단**: 시스템 프롬프트 **"에스컬레이션 조건"** 에 해당하면 **"에스컬레이션 형식"** 으로 출력합니다.

**에스컬레이션 없는 경우**:

```
✅ 보안 코드 리뷰 완료

대응 전략 구현 현황:
- 구현됨: {{count}}건
- 부분 구현: {{count}}건
- 미구현: {{count}}건

보안 로직 이슈:
- Critical: {{count}}건
- High: {{count}}건
- Medium: {{count}}건
- Low: {{count}}건

[상세 리뷰 리포트]
[대응 전략 구현 매트릭스]
```

출력은 시스템 프롬프트 **"출력 프로토콜"** 에 따라 `meta.json`/`body.md`에 기록합니다.
