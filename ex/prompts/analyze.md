# 의존성/아키텍처 분석 프롬프트

## 입력

```
scan 에이전트 출력: {{scan_output}}
detect 에이전트 출력: {{detect_output}}
```

## 지시사항

시스템 프롬프트에 정의된 역할과 규칙에 따라 `{{scan_output}}`과 `{{detect_output}}`을 기계적으로 분석하세요. 사용자에게 질문하지 말고 결과만 출력합니다.

### 경량/중량 모드 확인

`{{scan_output}}`의 `depth_mode.mode`를 확인합니다. 시스템 프롬프트 **"적응적 깊이"** 절에 따라 분석 범위를 조정합니다:

- **경량 모드**: Step 1 → Step 7로 건너뛰기 (디렉토리 기반 컴포넌트 분류만)
- **중량 모드**: Step 1 ~ Step 7 전체 수행

---

### Step 1: 컴포넌트 식별 (공통)

`{{scan_output}}`의 디렉토리 구조와 파일 분류를 기반으로 논리적 컴포넌트를 식별합니다. 시스템 프롬프트 **"컴포넌트 경계 추론 → 추론 규칙"** 및 **"컴포넌트 유형 분류"** 표에 따라 유형을 결정하고, 핵심 책임을 디렉토리 이름·파일 내용·export 패턴에서 추론한 뒤 `CM-001`부터 순차 ID를 부여합니다.

**경량 모드**: 여기서 간략 의존성 요약(매니페스트 기반 외부 의존만)만 추가하고 Step 7로 건너뜁니다.

---

### Step 2: Import 분석 (중량 모드)

`{{detect_output}}`의 언어 정보를 참조하여, 시스템 프롬프트 **"Import 분석 → 언어별 Import 구문"** 표에 따라 각 소스 파일의 import 구문을 파싱하고 내부/외부 의존을 분리합니다.

### Step 3: 의존성 그래프 구축 (중량 모드)

시스템 프롬프트 **"의존성 그래프 구축"** 절차에 따라 파일 단위 import를 디렉토리(컴포넌트) 단위로 집계하고, 컴포넌트 간 의존 방향과 역참조(dependents), 외부 의존을 도출합니다.

### Step 4: API 표면 식별 (중량 모드)

시스템 프롬프트 **"API 표면 식별"** 표의 탐지 패턴을 사용하여 각 컴포넌트가 외부에 노출하는 인터페이스를 탐지합니다. 각 API에 대해 경로/메서드를 가능한 범위에서 기록합니다.

### Step 5: 아키텍처 스타일 추론 (중량 모드)

시스템 프롬프트 **"아키텍처 스타일 추론"** 표의 증거 패턴에 따라 스타일을 추론하고, **"계층 구조 탐지"**, **"통신 패턴 탐지"** 절을 참조해 계층 구조와 통신 패턴을 도출합니다. 추론된 스타일에 대해 2개 이상의 증거를 제시합니다.

### Step 6: 횡단 관심사 및 순환 의존성 (중량 모드)

시스템 프롬프트 **"횡단 관심사 탐지"** 표의 패턴으로 횡단 관심사를 탐지하고, **"순환 의존성 탐지"** 절에 따라 순환 참조를 식별합니다. 발견 시 순환 경로와 해소 방향을 제안합니다.

### Step 7: 결과 출력

시스템 프롬프트 **"산출물 구조"**에 맞춰 YAML로 출력하고, **"출력 프로토콜"**에 따라 `meta.json`/`body.md`에 기록합니다.

경량 모드의 경우 다음 축약 형태로 출력합니다:

```yaml
components:
  - id: CM-001
    name: <이름>
    path: <경로>
    type: <유형>
    responsibility: <책임 — 디렉토리 기반 추론>
    dependencies_external: [<매니페스트 기반>]

architecture_inference:
  architecture_style: "분석 생략 (경량 모드)"
  style_evidence: []
```
