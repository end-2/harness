# 구조 스캔 프롬프트

## 입력

```
프로젝트 루트 경로: {{project_root}}
추가 제외 패턴: {{exclude_patterns}}
깊이 모드 오버라이드: {{depth_override}}
```

## 지시사항

시스템 프롬프트에 정의된 역할과 규칙을 따라 `{{project_root}}`를 기계적으로 분석하세요. 사용자에게 질문하지 말고 결과만 출력합니다.

### Step 1: 제외 패턴 확정

1. `{{project_root}}/.gitignore`를 파싱하여 무시 패턴 수집
2. 시스템 프롬프트 **".gitignore 패턴 해석 → 기본 제외 패턴"**을 병합 적용
3. `{{exclude_patterns}}`가 있으면 추가
4. 최종 패턴을 `ignored_patterns`에 기록

### Step 2: 디렉토리 트리 구축

- `{{project_root}}`부터 재귀 탐색, 제외 패턴 건너뛰기
- 시스템 프롬프트 **"토큰 효율적 트리 표현 → 축약 규칙"** 적용

### Step 3: 파일 분류

시스템 프롬프트 **"파일 분류 기준"** 표에 따라 6개 카테고리로 분류합니다. 하나의 파일이 다수 카테고리에 해당할 수 있으면 가장 구체적인 것을 선택합니다 (예: `jest.config.ts` → config).

### Step 4: 진입점 식별

시스템 프롬프트 **"진입점 식별 규칙"** 표의 패턴을 탐색하여 각 진입점의 경로와 역할을 기록합니다.

### Step 5: 설정 파일 매핑

시스템 프롬프트 **"설정 파일 매핑"** 표에 따라 경로·역할·카테고리를 기록합니다.

### Step 6: 적응적 깊이 판별

시스템 프롬프트 **"적응적 깊이 판별"** 기준으로 경량/중량 모드를 결정합니다. `{{depth_override}}`가 지정된 경우 자동 판별을 무시합니다. 판별 근거를 정량적으로 기록합니다:

```
depth_mode:
  mode: <lightweight | heavyweight>
  evidence:
    file_count: <소스 파일 수>
    language_count: <언어 수>
    framework_count: <프레임워크 수>
    max_directory_depth: <최대 디렉토리 깊이>
```

### Step 7: 디렉토리 규칙 탐지 (중량 모드만)

시스템 프롬프트 **"디렉토리 규칙 탐지"** 절의 패턴을 탐지하여 기록합니다.

### Step 8: 산출물 출력

시스템 프롬프트 **"산출물 구조"**에 맞춰 YAML로 출력하고, **"출력 프로토콜"**에 따라 `meta.json`/`body.md`에 기록합니다.
