# 요구사항 도출 출력 예시

> 예시 1 (고수준 입력: "사내 직원들이 사용할 휴가 관리 시스템")에 대한 multi-turn 대화 후 산출물

## 요구사항 후보

| ID | 분류 | 제목 | 우선순위 | 수용 기준 | 출처 |
|----|------|------|----------|-----------|------|
| FR-001 | functional/auth | 사내 SSO 로그인 | Must | - 사내 SSO 인증으로 로그인 성공 시 대시보드로 이동<br>- 미인증 사용자는 로그인 페이지로 리다이렉트 | 사용자 응답 (Turn 2) |
| FR-002 | functional/leave-request | 휴가 신청 | Must | - 연차/반차/병가/특별휴가 유형을 선택하여 신청 가능<br>- 시작일, 종료일, 사유를 입력하여 신청<br>- 신청 후 상태가 '대기중'으로 표시 | 사용자 응답 (Turn 1, 3) |
| FR-003 | functional/approval | 휴가 승인/반려 | Must | - 팀장은 팀원의 휴가 신청을 승인/반려 가능<br>- 반려 시 사유 입력 필수<br>- 승인/반려 시 신청자에게 이메일 알림 발송 | 사용자 응답 (Turn 3) |
| FR-004 | functional/calendar | 팀 휴가 캘린더 | Should | - 팀원들의 휴가 일정이 캘린더 뷰로 표시<br>- 월별/주별 보기 전환 가능 | 사용자 응답 (Turn 4) |
| FR-005 | functional/balance | 잔여 휴가 조회 | Must | - 유형별 총 휴가일수, 사용일수, 잔여일수 표시<br>- 연초에 연차가 자동 부여 (15일 기본, 근속 연수별 가산) | 사용자 응답 (Turn 2, 5) |
| FR-006 | functional/notification | 알림 | Should | - 승인/반려 시 이메일 알림<br>- 휴가 시작 전일 리마인더<br>- 팀장에게 미처리 신청 건 알림 (일 1회) | 사용자 응답 (Turn 6) + 에이전트 제안 |
| FR-007 | functional/admin | 관리자 기능 | Should | - HR 담당자가 전 직원 휴가 현황 조회 가능<br>- 휴가 정책(유형, 일수) 설정 가능<br>- 연간 휴가 사용 통계 리포트 생성 | 에이전트 제안 → 사용자 확인 (Turn 7) |
| NFR-001 | non-functional/performance | 응답 시간 | Should | - 페이지 로드 시간 2초 이내<br>- 캘린더 뷰 렌더링 3초 이내 | 에이전트 질문 → 사용자 응답 (Turn 8) |
| NFR-002 | non-functional/availability | 가용성 | Should | - 업무 시간(09:00-18:00) 내 99.5% 가용성<br>- 야간 유지보수 허용 | 에이전트 질문 → 사용자 응답 (Turn 8) |

## 제약 조건 후보

| ID | 유형 | 제목 | 설명 |
|----|------|------|------|
| CON-001 | technical | 사내 SSO 연동 | 기존 사내 SAML 2.0 기반 SSO와 연동 필수 |
| CON-002 | business | 직원 수 규모 | 전체 사용자 약 500명, 동시 접속 최대 100명 |
| CON-003 | environmental | 사내 클라우드 | 사내 프라이빗 클라우드(OpenStack)에 배포 |
| CON-004 | business | 런칭 시기 | 내년 1월까지 런칭 (약 3개월) |
| CON-005 | regulatory | 개인정보 | 직원 개인정보(병가 사유 등) 처리 시 개인정보보호법 준수 |

## 품질 속성 힌트

| 속성 | 사용자 언급 내용 |
|------|-----------------|
| usability | "직원들이 별도 교육 없이 바로 사용할 수 있어야 합니다" (Turn 4) |
| availability | "업무 시간에는 반드시 사용 가능해야 합니다" (Turn 8) |
| security | "병가 사유 같은 민감 정보는 팀장만 볼 수 있어야 합니다" (Turn 6) |
| maintainability | "HR 정책이 매년 바뀌므로 설정으로 관리하고 싶습니다" (Turn 7) |

## 미해결 질문

- [ ] 모바일 접근이 필요한가? (웹 반응형으로 충분한지, 네이티브 앱이 필요한지)
- [ ] 기존에 사용 중인 휴가 관리 시스템이 있는가? 데이터 마이그레이션이 필요한가?
- [ ] 대리 승인 기능이 필요한가? (팀장 부재 시)
