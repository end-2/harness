# 다이어그램 생성 프롬프트

## 입력

```
컴포넌트 구조: {{component_structure}}
기술 스택: {{technology_stack}}
적응적 깊이 모드: {{mode}}  (lightweight | heavyweight)
```

## 지시사항

시스템 프롬프트에 정의된 역할과 규칙을 따라 확정된 설계를 Mermaid 코드로 시각화하세요.

### Step 1: 다이어그램 계획

시스템 프롬프트 **"적응적 깊이"**에 따라 `{{mode}}`에 맞는 다이어그램 목록을 결정하고 사용자에게 제시합니다:

```
다음 다이어그램을 생성하겠습니다:
1. [C4 Context] {{시스템명}} 컨텍스트
2. [C4 Container] {{시스템명}} 컨테이너 구조
3. [Sequence] {{주요 흐름}} 시퀀스

추가하거나 제외할 다이어그램이 있나요?
```

### Step 2: C4 Context 다이어그램 생성

시스템 프롬프트 **"다이어그램 유형별 가이드 → C4 Context 다이어그램"**의 포함 요소와 Mermaid 코드 패턴을 따릅니다. 사용자 유형은 FR에서, 외부 시스템은 제약 조건·인터페이스에서 도출합니다.

### Step 3: C4 Container 다이어그램 생성 (중량 모드)

시스템 프롬프트 **"다이어그램 유형별 가이드 → C4 Container 다이어그램"**을 참조합니다. `component_structure`의 각 항목을 노드로 변환하되, `type`에 따라 노드 형태를 결정합니다:

- `service` → 직사각형
- `store` / `database` → 원통형 `[(" ")]`
- `queue` → 이중 중괄호 `{{ }}`
- `gateway` → 사다리꼴 또는 직사각형
- `ui` → 직사각형 (별도 서브그래프)

`interfaces`에서 통신 방향·프로토콜을, `technology_stack`에서 기술명을 매핑해 노드 레이블에 포함합니다.

### Step 4: 시퀀스 다이어그램 생성 (중량 모드)

시스템 프롬프트 **"다이어그램 유형별 가이드 → 시퀀스 다이어그램"**의 흐름 선정 기준과 Mermaid 패턴을 따라 주요 흐름을 시각화합니다.

### Step 5: 사용자 피드백 및 수정

생성된 다이어그램을 제시하고 피드백을 받아 수정·확정합니다.

### Step 6: 산출물 기록

시스템 프롬프트 **"산출물 구조"** YAML 형식과 **"스타일 가이드"**(일관된 표기, 가독성, Mermaid 호환성)를 따르며, **"출력 프로토콜"**에 따라 `meta.json`/`body.md`에 기록합니다.
