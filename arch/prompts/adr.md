# ADR 생성 프롬프트

## 입력

```
아키텍처 결정 요약: {{architecture_decisions}}
```

## 지시사항

시스템 프롬프트에 정의된 역할과 규칙을 따라 `{{architecture_decisions}}`를 Michael Nygard 형식 ADR로 기록하세요.

### Step 1: ADR 대상 결정 식별

시스템 프롬프트 **"생성 프로세스 → 단계 1: 결정 분류"**의 "ADR 작성 필수/권고" 기준에 따라 각 결정의 ADR 작성 여부를 판단하고 목록화합니다:

```
ADR 작성 대상:
1. AD-001: {{제목}} → ADR-001
2. AD-002: {{제목}} → ADR-002
3. AD-003: {{제목}} → (ADR 생략 — 사유: ...)
```

### Step 2: ADR 작성

시스템 프롬프트 **"ADR 형식"** 템플릿과 **"생성 프로세스 → 단계 2: ADR 초안 작성"** 절차에 따라 각 ADR 초안을 작성합니다.

**체크리스트**:
- [ ] RE 근거(`re_refs`)가 컨텍스트에 포함되어 있는가
- [ ] `alternatives_considered`가 대안 비교표로 변환되었는가
- [ ] `trade_offs`가 긍정적/부정적 결과로 구분되었는가
- [ ] 기각 사유가 객관적으로 작성되었는가
- [ ] ADR 간 관계(supersedes, amends, relates-to)가 설정되었는가

### Step 3: RE 참조 연결

시스템 프롬프트 **"생성 프로세스 → 단계 3: RE 참조 연결"**에 따라 각 ADR 컨텍스트에 RE 산출물 ID(NFR/CON/QA)를 명시합니다.

### Step 4: 관계 설정

시스템 프롬프트 **"생성 프로세스 → 단계 4: 관계 설정"**에 따라 ADR 간 관계(supersedes/amends/relates-to)를 설정합니다.

### Step 5: 사용자 확인

ADR 목록을 사용자에게 제시하고 확인을 요청합니다:

```
다음 ADR을 작성했습니다:

1. ADR-001: {{제목}} — {{한 줄 요약}}
2. ADR-002: {{제목}} — {{한 줄 요약}}

각 ADR의 내용을 확인해주세요. 수정이 필요한 부분이 있으면 말씀해주세요.
```

### Step 6: 산출물 기록

시스템 프롬프트 **"출력 프로토콜"**에 따라 `meta.json`/`body.md`에 기록합니다.
