# 아키텍처 설계 프롬프트

## 입력

```
요구사항 명세: {{requirements_spec}}
제약 조건: {{constraints}}
품질 속성 우선순위: {{quality_attribute_priorities}}
```

## 지시사항

시스템 프롬프트에 정의된 역할과 규칙을 따라 RE 산출물을 분석하고, 사용자와 기술적 맥락 대화를 거쳐 시스템 아키텍처를 설계하세요.

### Step 1: RE 산출물 분석

시스템 프롬프트 **"RE 산출물 해석 방법"**에 따라 다음을 도출합니다:
- 아키텍처 드라이버 (품질 속성 상위 priority + `flexibility: hard` 제약)
- 컴포넌트 후보 (FR category/dependencies, NFR 횡단 관심사)
- 적응적 깊이 모드 판별 — 시스템 프롬프트 **"적응적 깊이"** 기준 적용

분석 결과를 사용자에게 제시합니다:

```
RE 산출물 분석 결과:
- 요구사항: FR {{count}}개, NFR {{count}}개
- 제약 조건: {{count}}개 (hard: {{count}}, soft: {{count}}, negotiable: {{count}})
- 품질 속성: {{count}}개
- → [경량/중량] 모드로 진행합니다

핵심 아키텍처 드라이버:
1. {{quality_attribute}} (priority {{n}}): {{metric}}
2. {{constraint}} (hard): {{description}}
...
```

### Step 2: 기술적 맥락 질문

시스템 프롬프트 **"기술적 맥락 도출"**의 4개 영역(팀 역량, 기존 인프라, 비용 제약, 기존 코드베이스)을 참고하여 한 번에 3-4개 질문으로 제한해 사용자에게 확인합니다. **제약 조건에서 이미 확인된 정보는 재질문하지 마세요.**

### Step 3: 아키텍처 스타일 결정

시스템 프롬프트 **"아키텍처 스타일 결정"** 패턴 카탈로그를 참조해 후보 스타일 2-3개를 선정하고 RE 품질 속성·제약 조건 기준으로 비교 분석한 뒤 추천합니다:

```
아키텍처 스타일 비교:

| 기준 | {{스타일 A}} | {{스타일 B}} | {{스타일 C}} |
|------|-------------|-------------|-------------|
| {{품질속성 1}} 충족 | ... | ... | ... |
| {{제약조건 1}} 준수 | ... | ... | ... |
| 팀 역량 적합성 | ... | ... | ... |
| 복잡도 | ... | ... | ... |

→ {{스타일 A}}를 추천합니다. 이유: ...
```

### Step 4: 컴포넌트 설계 (중량 모드)

경량 모드에서는 레이어/디렉토리 구조 가이드로 대체합니다. 중량 모드에서는 다음을 정의하세요:

1. **컴포넌트 식별**: FR category 기반 주요 컴포넌트 도출
2. **책임 할당**: 각 컴포넌트에 FR/NFR 매핑 (`re_refs`)
3. **인터페이스 정의**: 컴포넌트 간 통신 방식(프로토콜, 방향)
4. **의존 관계 설정**: 순환 의존 금지

### Step 5: 기술 스택 결정

각 기술 카테고리(language/framework/database/messaging/infra/tool)에 대해 선택과 근거를 다음 순서로 제시합니다:
1. RE 제약 조건 준수 여부 (`constraint_ref`)
2. 팀 기존 경험/역량
3. 품질 속성 충족 능력
4. 커뮤니티 성숙도/생태계

### Step 6: 설계 초안 제시 및 피드백

시스템 프롬프트 **"산출물 구조"**에 맞춰 초안(아키텍처 결정 요약, 컴포넌트 구조, 기술 스택)을 제시하고 피드백을 요청합니다:

```
피드백을 주세요:
1. 아키텍처 스타일이 적절한가요?
2. 컴포넌트 분할에 동의하시나요?
3. 기술 스택 중 변경하고 싶은 것이 있나요?
4. 추가로 고려해야 할 기술적 제약이 있나요?
```

### Step 7: 피드백 반영 및 확정

사용자 피드백을 반영하여 최종 산출물을 확정합니다.

### Step 8: 산출물 기록

시스템 프롬프트 **"산출물 구조"** YAML 스키마에 맞춰 작성하고, **"출력 프로토콜"**에 따라 `meta.json`/`body.md`에 기록합니다.
